D. Content

The note says inference is becoming memory-constrained.

The author argues that for AI inference, especially decode, the bottleneck is not arithmetic throughput but memory bandwidth and capacity.

Prefill and decode are different workloads.

Prefill processes the prompt/context in parallel and is compute-heavy. Decode generates tokens one at a time and repeatedly reads model weights and KV-cache state, making it memory-bound.

Agentic workloads make KV cache a bigger problem.

Long coding, research, and computer-use agents hold large repositories, tool outputs, traces, and plans in context for long sessions. That turns KV cache into a dominant HBM consumer.

The visible article starts the optimization-stack discussion but cuts off.

The preview introduces continuous batching and says the long tail of enterprises has unused efficiency potential, but the rest of the techniques are behind the subscriber boundary in the current browser profile.

H. Appendix

Decode explains why memory matters.

During decode, the model produces one token at a time and repeatedly accesses weights and KV-cache state. That can leave expensive compute underutilized while waiting on memory.

Agents worsen the constraint.

Long-running agents keep far more context alive than short chatbot sessions. That makes memory capacity and batching economics more important.

The investment question is not only more chips.

If memory per token falls, the same installed base can serve more demand. That may help hyperscalers and inference platforms while complicating simple HBM-demand extrapolation.
